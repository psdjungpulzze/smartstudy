# Phase 1: Discovery

## Overview

The Discovery phase focuses on understanding the problem space, gathering requirements, and validating the product concept before committing to development.

## Objectives

- Define clear problem statement
- Identify stakeholders and users
- Gather and prioritize requirements
- Conduct competitive analysis
- Validate assumptions with research

## Duration

Typical: 1-2 weeks (scale based on project complexity)

---

## Deliverables

### 1. Problem Statement Document
- Clear articulation of the problem
- Impact and scope definition
- Success metrics

### 2. Stakeholder Analysis
Template: [stakeholder-analysis.md](./stakeholder-analysis.md)
- All stakeholders identified
- Influence and interest mapped
- Communication plan defined

### 3. Requirements Document
Template: [requirements-template.md](./requirements-template.md)
- Functional requirements
- Non-functional requirements
- Constraints and assumptions

### 4. User Stories
Template: [user-story-template.md](./user-story-template.md)
- As a [user], I want [goal] so that [benefit]
- Acceptance criteria for each story

---

## Process

### Step 1: Problem Definition

1. Conduct stakeholder interviews
2. Document pain points and challenges
3. Define the problem statement
4. Identify success metrics

**Output**: Problem statement document

### Step 2: Stakeholder Analysis

1. List all stakeholders
2. Categorize by influence and interest
3. Define communication strategy
4. Identify decision makers

**Output**: Stakeholder map

### Step 3: Requirements Gathering

1. Conduct discovery sessions
2. Document functional requirements
3. Define non-functional requirements
4. Identify constraints

**Output**: Requirements document

### Step 4: User Research

1. Define user personas
2. Map user journeys
3. Identify pain points
4. Document jobs-to-be-done

**Output**: User personas and journey maps

### Step 5: Competitive Analysis

1. Identify competitors
2. Analyze features and positioning
3. Identify differentiation opportunities
4. Document market gaps

**Output**: Competitive analysis report

## Exit Criteria

Before moving to Planning phase:

1. ✅ Problem statement approved by stakeholders
2. ✅ Requirements prioritized (MoSCoW or similar)
3. ✅ Initial scope defined
4. ✅ Risks identified
5. ✅ Go/no-go decision made
6. ✅ Every requirement has a unique ID (FR-XXX) and acceptance criteria
7. ✅ Every requirement (FR-XXX) maps to at least one user story (US-XXX)
8. ✅ Every user story references its parent requirement(s) via FR-XXX
9. ✅ Requirements Coverage Matrix has Req ID, Title, Priority, and User Stories columns filled
10. ✅ No raw requirement left unaddressed or downgraded without justification

## AI Collaboration Tips

### Effective Prompts

```
"Help me define the problem statement for [project].
Consider: Who is affected? What is the impact?
What happens if we don't solve this?"
```

```
"Analyze these requirements and identify:
1. Missing edge cases
2. Potential conflicts
3. Unclear acceptance criteria"
```

```
"Generate user stories for [feature] covering:
- Primary user flow
- Error scenarios
- Edge cases"
```

```
"Create a stakeholder analysis for a [type] project.
Include: influence level, interest level, communication needs"
```

```
"What questions should I ask stakeholders about [topic]?"
```
